# BETWEEN STATES
### A Live Performance XR Platform

A mobile-first XR system that transforms live music into a shared, embodied, and identity-driven visual experience.

---

## 1. PROJECT OVERVIEW

Between States is a live performance XR platform designed for festival, rave, and concert environments.

Audience members access the experience via QR code and activate a mobile AR layer that responds to:
- live audio from the environment
- device movement (tilt, shake)
- camera input (face tracking)

The result is a dynamic, glitch-based visual system that transforms passive spectators into active participants in the performance.

This is not a second screen experience.

This is an extension of the stage.

---

## 2. CORE PREMISE

Between States is a mobile XR experience that transforms live music into an interactive, identity-driven visual system for audiences by using audio-reactive and embodied AR.

The system reframes identity as:
- fluid
- unstable
- continuously transforming

Glitch is used as a behavioral language of resistance:
- refusing fixed form
- disrupting visual coherence
- expressing identity as interference rather than object

---

## 3. EXPERIENCE (USER JOURNEY)

### Entry
- User scans QR code at a live event
- Web-based XR experience loads instantly
- Permissions requested (camera, microphone, motion)

---

### Modes of Interaction

#### Rear Camera Mode (Shared Experience)
- User points device at stage or environment
- Visuals overlay physical space
- Audio from environment drives transformation

#### Front Camera Mode (Identity Mode)
- Face tracking activates
- Glitch visuals attach to the face
- Identity appears, distorts, collapses

---

### Behavioral System

The experience moves through states:

1. **World in Context**
   - Minimal visual intervention
   - System is latent

2. **Emergence**
   - Glitch elements begin to appear
   - Forms suggest identity

3. **Distortion**
   - Audio + motion intensify visuals
   - Fragmentation increases

4. **Collapse / Becoming**
   - Identity dissolves into abstraction
   - No stable form is achieved

---

## 4. XR DIFFERENTIATION

This project requires XR because:

- The environment directly drives the visuals
- The user’s body influences the system
- The experience is site-specific and context-aware

### Key XR Attributes

1. **Embodied Interaction**
   - Device movement alters visual output
   - User physically engages with system

2. **Face / Body Anchoring**
   - Identity-linked visuals via front camera
   - Self becomes the canvas

3. **Site-Specific Meaning**
   - Visuals driven by live audio environment
   - Experience cannot be replicated outside context

## 5. PHYSIOLOGICAL + MULTISENSORY DESIGN

The experience is designed to feel:

- performative
- unstable
- undefined

### Sensory Mapping

- **Sound intensity**
  → controls opacity, contrast, and presence

- **Device movement**
  → increases instability and distortion

- **Proximity / crowd energy (implicit)**
  → amplifies immersion through environment

### Accessibility Considerations

- Visual intensity scales with sound
- Quiet environments reduce stimulation
- Users can disengage by lowering device

## 6. ETHICAL + CULTURAL FRAMEWORK

Between States explores:

- queer identity as fluid and constructed
- resistance to binary classification
- visibility vs surveillance of queer bodies

The system acts as a metaphor for:

- identity that cannot be cleanly rendered
- bodies that resist categorization
- presence that shifts across contexts

This aligns with media theory concepts:
- “the medium is the message” (McLuhan)
- technology shaping perception of identity

## 8. TECHNICAL SYSTEM (INFORMATION ARCHITECTURE)

### Devices + Nodes

- Participant smartphones (primary interface)
- Live audio environment (DJ / stage sound)
- QR code entry system
- Mobile network (Wi-Fi / cellular)
- Front-facing camera (face tracking)

---

### Software Stack

- Hydra → real-time generative visuals
- p5.js → audio + interaction mapping
- WebAR → browser-based XR delivery
- Vite → build + deployment

---

### Data Flow

Audio Input → FFT → State System → Visual Mapping → AR Overlay

Device Motion → Distortion Parameters

Face Detection → Identity Anchoring
